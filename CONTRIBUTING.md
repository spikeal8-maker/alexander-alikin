# Как вносить изменения

`main` — опубликованная и проверенная версия. Новые материалы, изменения
биографии и технические правки проходят через Pull Request.

## Ветки

- `content/<slug>` — новый текст или обновление;
- `feature/<slug>` — новая функция;
- `fix/<slug>` — исправление;
- `research/<slug>` — исследование без немедленной публикации.

## Контентный Pull Request обязан содержать

1. Что изменено.
2. Почему материал нужен.
3. На какие источники он опирается.
4. Какой у утверждений уровень доказательности.
5. Какие сведения требуют проверки.
6. Какие страницы затрагиваются.

## Запрещено коммитить

- пароли, токены и ключи API;
- договоры, платёжные данные и персональные документы;
- сведения о несовершеннолетних без разрешения;
- материалы со статусом `private`;
- неподтверждённые финансовые показатели как установленный факт.

## Проверка

```bash
npm run check
npm run build
```
