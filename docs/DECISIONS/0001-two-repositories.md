# ADR-0001: разделить публичный сайт и приватный источник

- Статус: принято
- Дата: 2026-07-19

## Контекст

Исходный архив содержит одновременно публичные идеи, рабочие черновики, личные сведения и утверждения разной степени подтверждённости. Публикация сайта из одного общего хранилища создаёт риск случайной утечки и смешивает редакционную работу с production-кодом.

## Решение

Используются два репозитория:

- `alexander-alikin` — публичный код и утверждённый контент;
- `alexander-alikin-vault` — приватный источник, реестры, доказательства и подготовка.

Между репозиториями нет прямого автоматического импорта. Публикация проходит через экспортный пакет и Pull Request.

## Последствия

Положительные: явная граница публичности, прозрачный diff, переносимость сайта, история решений. Отрицательные: требуется дисциплина редакционного экспорта и двойная структура файлов.
