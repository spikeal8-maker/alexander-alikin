# Редакционный процесс

1. **Входящий материал.** Заметка, документ, расшифровка, фото, ссылка, новость.
2. **Классификация.** Тип, аудитория, чувствительность, публичность, утверждения.
3. **Проверка.** Формулировка, источник, дата, владелец, статус и разрешение.
4. **Редактура.** Контекст, роль Александра, факт/мнение/вывод, связи.
5. **Pull Request.** В public переносятся только готовые файлы и медиа.
6. **CI.** Проверяются schema, TypeScript, сборка и запрещённые статусы.
7. **Публикация.** Merge в `main` запускает GitHub Pages.
8. **Пересмотр.** Материал обновляется при изменении роли, проекта или источника.

## Редакционная периодичность

- «Сейчас» — минимум раз в месяц;
- новости — по факту;
- короткая мысль — ориентир раз в неделю;
- история или разбор — два раза в месяц;
- системная статья — один раз в месяц.
