# Итог архитектуры foundation-версии

Система состоит из двух независимых репозиториев и контролируемого редакционного шлюза.

## Публичная платформа

`alexander-alikin` содержит код, утверждённые материалы и документацию. `main` соответствует опубликованной версии. Astro собирает статические страницы, GitHub Actions выполняет проверки и развёртывание.

## Приватный источник

`alexander-alikin-vault` содержит исходные заметки, инвентари, реестры утверждений, источников и разрешений, а также редакционные стадии. Он не подключён к production и не является хранилищем секретов.

## Публикационный шлюз

Материал проходит классификацию, фактчек, права, редактуру, решение Александра, локальный экспорт, отдельную ветку и Pull Request. Только после этого он может попасть в `main`.

## Цель

Создать не одноразовый лендинг, а долгосрочную цифровую биографию, авторское медиа и профессиональный центр доверия, который можно развивать через прозрачные изменения в GitHub.
