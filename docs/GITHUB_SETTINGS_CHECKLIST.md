# Настройки GitHub после слияния

## Pages

1. `Settings → Pages`.
2. Source: `GitHub Actions`.
3. Проверить первый deploy.
4. Перед подключением домена подтвердить его в настройках аккаунта.
5. Затем добавить домен в Pages и настроить DNS.

## Защита `main`

- изменения через Pull Request;
- обязательный status check `validate`;
- запрет force-push и удаления;
- squash merge;
- удаление head-ветки после merge.

Пока автор один, обязательный внешний review можно не включать, но CI должен
оставаться обязательным.

## Issues и Discussions

Issues — задачи и исправления. Discussions — позже, для обсуждения материалов.
Секреты и персональные данные не публикуются в Issues.
