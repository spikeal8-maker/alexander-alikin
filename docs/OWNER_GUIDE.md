# Руководство владельца

## Что можно делать без программирования

1. Создать новую ветку `content/<slug>`.
2. Скопировать подходящий Markdown-шаблон.
3. Заполнить метаданные и текст.
4. Добавить подготовленные изображения в `public/images`.
5. Открыть Pull Request.
6. Просмотреть автоматические проверки и изменения.
7. После проверки слить Pull Request в `main`.

## Что не следует делать напрямую

- загружать приватный архив в публичный репозиторий;
- редактировать `main` для сложного материала без Pull Request;
- публиковать цифры без proof level и реестра;
- хранить токены в файлах;
- размещать оригинальные фотографии с лишними метаданными;
- подключать форму или аналитику без отдельного решения.

## Как обновить «Сейчас»

Обновляется структурированный файл текущих направлений. Формулировки должны отвечать на вопросы: что создаётся, для кого, на каком этапе и где посмотреть подтверждение.

## Как исправить факт

Исправление проходит обычный Pull Request. В описании указываются старая формулировка, новая формулировка, причина, источник и затронутые страницы.

## Как снять материал

Материал не обязательно удалять из истории. Его можно перевести в архивный статус, убрать из индекса и сохранить объяснение, почему он больше не отражает текущую позицию.
