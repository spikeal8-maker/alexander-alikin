# Приватность и безопасность

## Классы

- `Public` — разрешено публиковать.
- `Internal` — только редакционная работа.
- `Sensitive` — финансовые, медицинские, семейные и договорные сведения.
- `Secret` — пароли, токены, API-ключи и приватные ключи.

## Правила

- `Sensitive` и `Secret` не попадают в public;
- секреты не хранятся обычными файлами даже в private;
- используются GitHub Actions Secrets;
- фотографии детей требуют разрешения;
- частная переписка не становится публичной цитатой автоматически;
- секрет, попавший в commit, считается скомпрометированным и заменяется.

На первом этапе сайт не хранит данные собственной формой: контакт идёт через
существующие каналы.
