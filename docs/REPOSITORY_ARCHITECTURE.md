# Архитектура двух репозиториев

## Публичный `alexander-alikin`

Содержит код сайта, проверенную биографию, опубликованные материалы,
публичные карточки проектов, разрешённые изображения, редакционные правила
и CI/CD.

Не содержит исходный личный архив, договоры, финансовые выгрузки, документы с
персональными данными, секреты и материалы о несовершеннолетних без разрешения.

## Приватный `alexander-alikin-vault`

Содержит исходный Obsidian-архив, стратегическое досье, реестр заметок,
реестр фактов и доказательств, черновики, материалы на проверку и оригиналы
источников.

## Поток данных

```text
источник / новая идея
        ↓
vault / inbox
        ↓
классификация чувствительности
        ↓
проверка фактов
        ↓
редакционный черновик
        ↓
approved-for-publication
        ↓
Pull Request в публичный репозиторий
        ↓
CI: схема, типы, сборка
        ↓
review → merge → GitHub Pages
```

## Запрет на прямую синхронизацию

`public_use: true` означает только предварительное разрешение. Материал всё
равно должен пройти редактуру, проверку и отдельное утверждение.
