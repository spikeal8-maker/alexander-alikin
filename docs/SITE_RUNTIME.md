# Как работает сайт

## Сборка

Контент хранится в Markdown/MDX и проверяется схемами Astro content collections. При Pull Request запускаются проверки структуры и сборка. После слияния в `main` GitHub Actions собирает статический сайт и передаёт результат GitHub Pages.

## Маршруты

- `/` — редакционная главная;
- `/about/` — каноническая биография;
- `/projects/` — каталог проектов и отдельные страницы;
- `/stories/` — человеческие истории;
- `/thoughts/` — короткие авторские идеи;
- `/articles/` — системные материалы;
- `/news/` — фактические обновления;
- `/now/` — текущая деятельность;
- `/second-brain/` — метод работы со знаниями;
- `/collaboration/` — способы сотрудничества;
- `/contacts/` — канонические контакты;
- `/search/` — поиск по опубликованному массиву;
- `/rss.xml` и `/sitemap-index.xml` — машинные каналы обновлений и индексации.

## Главная страница

Главная не редактируется как единая рекламная простыня. Она собирает актуальные данные из контентных коллекций: текущие направления, избранные проекты, новые истории, мысли и новости. Поэтому новый материал может появиться на витрине без ручного изменения разметки.

## Связи

Материалы связываются темами, проектами и отношениями `related`. На следующем этапе из этих связей формируется семантическая навигация и публичная карта знаний.

## Ограничения статической архитектуры

Сайт не имеет собственного серверного хранилища персональных данных. Интерактивные формы, комментарии и подписка подключаются только отдельным решением после выбора правовой и технической модели. Базовый контакт ведёт на официальные каналы Александра.
