# Техническая архитектура

## Astro

Сайт генерируется статически. На выходе — HTML, CSS, JSON и RSS без постоянного
серверного приложения.

Преимущества для проекта:

- высокая скорость;
- простое размещение;
- низкая поверхность атаки;
- тексты хранятся как Markdown;
- история каждого материала находится в Git;
- сборка воспроизводится через GitHub Actions.

## Контентные коллекции

`src/content.config.ts` проверяет структуру проектов, историй, мыслей, новостей
и записей «Сейчас». Некорректный frontmatter должен остановить сборку.

## Поиск

На этапе 1 сервер поиска не нужен. При сборке создаётся `search-index.json`,
а браузер фильтрует опубликованные заголовки, описания и темы.

При росте до сотен материалов можно перейти на Pagefind или отдельный индекс,
не меняя модель контента.

## RSS и sitemap

RSS даёт независимый канал подписки. Sitemap формируется при сборке и помогает
поисковым системам обнаруживать страницы.

## SEO

BaseLayout формирует:

- canonical;
- description;
- Open Graph;
- JSON-LD Person/WebSite;
- языковой код;
- RSS discovery.

## GitHub Pages

До домена сайт использует:

```text
https://spikeal8-maker.github.io/alexander-alikin/
```

Поэтому `base` установлен в `/alexander-alikin`. При переходе на собственный
домен `SITE_URL` и `BASE_PATH` меняются без реорганизации контента.

## CI/CD

- PR: установка зависимостей, проверка контента, сборка;
- `main`: сборка официальным Astro Action и deploy-pages;
- публикация не зависит от локального компьютера.

## Будущие функции

- изображения с адаптивными размерами;
- медиакит;
- публичный граф знаний;
- комментарии через Discussions;
- аналитика;
- квалификационный бриф;
- ограниченный ИИ-поиск по опубликованной базе.
